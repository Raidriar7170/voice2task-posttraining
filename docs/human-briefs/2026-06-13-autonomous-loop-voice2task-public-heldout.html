项目阶段进展

当前目标是把 Voice2Task 的 public-sample contract learning 证据链收紧。此轮贡献是把“数据量不够或 held-out 覆盖不够”的假设做成可验证实验，并得到负结果：单纯小规模扩充 train exemplars + prompt policy 的 SFT-only 路线不够。

剩余风险和不能夸大的点

  • 不能声称模型真实能力约 70%，当前 strict evidence 不支持。
  • 不能把 JSON/schema valid 当作 contract recovery。
  • 不能声称私有语料泛化、生产可用、checkpoint/adapter release 或 live-browser benchmark 改善。
  • public 样本仍小，但足够否定“train split 过了就自动泛化”的说法。

推荐下一步

需要用户确认后开启一个很小的新 OpenSpec 阶段：诊断 SFT label/loss mask、目标 token 压力、prompt 长度与是否需要真正启用 DPO hard negatives。该阶段不应再直接盲目 A100 rerun。
