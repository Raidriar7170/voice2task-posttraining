本阶段做了什么

  • 重新做 A100 preflight，选择空闲 GPU 3，没有中断其他进程。
  • 在私有远端 root 下训练 a100-current-train-split-sft-retry adapter，使用当前 formal public train split 的 118 条 SFT rows。
  • 用新 adapter 对 dev/test 各 69 条样本做 prediction-only generation。
  • 用现有 strict evaluator 评估 dev/test，并只导入 public-safe evidence。

项目阶段进度

当前项目阶段目标仍是 evidence-backed post-training / evaluation baseline，而不是生产发布。此阶段把 readiness 证据推进为真实 A100 training + strict held-out evidence；剩余 gap 是解释 current retry 的 trade-off：dev exact 回退、confirmation 回退、test route 回退与 remaining slot residual。

推荐下一步

进入一个 bounded residual/trade-off diagnosis phase，先定位 current retry 的 confirmation/exact/route 回退和剩余 slot residual，再决定是否需要非常小的数据修补或停止为阶段性交付。
