如何回答“是不是数据泛化”

是，但更准确地说，是家族覆盖和 slot value 泛化问题，不是简单“数据量不够”。这次 12 条 targeted rows 已经让 train exact 变成 100%，dev/test 也从 0 起跳；所以后面要继续扩数据，也应该围绕失败家族和 slot value residual 做诊断式扩展，而不是直接上大规模混合训练或 DPO。

不要夸大的点

  • 不证明模型已经达到一开始想要的稳定语气和完整合同泛化。
  • 不发布 adapter/checkpoint，不声称 private corpus 泛化或生产可用。
  • 不把 soft slot F1 当主指标；主指标仍然是 strict contract_exact_match。
  • 不说明 prompt、evaluator、DPO 或大规模数据扩展已经有效。
