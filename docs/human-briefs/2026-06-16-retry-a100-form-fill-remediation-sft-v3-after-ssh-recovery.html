Voice2Task SFT v3 重试简报

observed, partial improvement

一句话结论：SFT v3 在当前 formal public held-out 上带来 dev exact/strict slot 明显改善和 test exact 小幅改善，但 dev safety recall 回退，因此只能算局部有效，不能宣称模型恢复或生产可用。

本阶段做了什么

  • 恢复 A100 连接后重新执行 preflight，选择空闲 GPU 运行。
  • 用当前 public train split 训练 SFT v3：114 rows，其中 21 条是 form-fill remediation / confirmation-marker 相关样本。
  • 用新 private adapter 跑 dev/test prediction-only，并用 strict evaluator 评估。
  • 只导入 public-safe evidence；未提交 adapter、checkpoint、raw log、private override、远端缓存或私有路径。

验证状态

  • 远端 SFT v3：training_completed，adapter 生成但不发布。
  • dev/test prediction：各 69 条，json_valid_rate=1.0。
  • evidence leak scan：ok=true，0 findings。

剩余风险

  • dev safety recall 从 0.6667 降到 0.5556，下一阶段应优先诊断 blocked_payment / safety 假阴性。
  • test strict slot_f1 基本未提升，说明 form-fill 样本扩展不能解决全部 slot 残差。
  • 这不是 production readiness、live browser benchmark、checkpoint release 或 private corpus generalization evidence。

推荐下一步

开一个很小的 bounded OpenSpec phase，专门诊断 SFT v3 后的 safety regression / blocked_payment 假阴性，而不是继续盲目加数据或再跑 DPO。
