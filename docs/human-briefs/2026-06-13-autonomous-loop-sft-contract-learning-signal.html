项目阶段进度

当前阶段目标来自 CONTEXT.md 和 README.md：Voice2Task Post-Training 的第一阶段是 speech-to-contract normalization，围绕 public sample、SFT/DPO 格式、contract metrics 和 public-safe reports 建证据链。仓库未明确定义 MVP/v1 标签，因此这里不新增里程碑名。

本阶段推进的是“为什么 7B SFT 没学稳 contract”的证据边界：它排除了一个本地结构性问题，即 public SFT 渲染文本缺少 assistant target span；但它也明确暴露了剩余缺口，真实 runtime label-mask 和 token-level 训练信号还没被验证。

剩余风险和不能夸大的点

  • 不能声称模型已经恢复，或真实能力约 70%。本阶段没有跑 prediction，也没有训练。
  • 不能把 assistant target span 存在等同于 runtime loss mask 正确。
  • 不能声称 private corpus generalization、checkpoint/adapter release、production readiness 或 live-browser benchmark 改善。
  • public sample 仍小；它适合做诊断和 public-safe evidence，不等于完整训练集结论。

需要用户确认后开启一个很小的新 OpenSpec 阶段：run-bounded-runtime-label-or-tiny-overfit-diagnostic。优先检查真实训练 tokenizer/collator 的 assistant-label mask、target token counts、prompt budget；必要时只做 1-3 条 public row 的 tiny-overfit 诊断。不要直接盲目扩数据或启动完整 A100 rerun。
